### 1.2 当前架构

```
用户浏览器 → http://122.51.56.50
                  │
            Nginx（:80）
            ├─ 静态文件（/usr/share/nginx/html）
            └─ 反向代理 → Node.js API（api:3000/server.js）
                               │
                           PostgreSQL 17（db:5432, Docker 容器）
```

| 组件 | 技术 | 部署方式 |
|------|------|---------|
| 前端 | 纯 HTML/CSS/JS（无框架） | Docker Nginx 容器 |
| 后端 | Node.js 自建 REST API（server.js） | Docker 容器 |
| 数据库 | PostgreSQL 17 | Docker 容器 |
| 服务器 | 腾讯云轻量应用服务器 Ubuntu 22.04 | 物理机 |

| `auth.js` | 160+ | 用户认证模块（已迁移到服务端登录） |

| `app.js` | 71 | Tab 路由、初始化、键盘事件 |

### 1.4 关键技术债

| 风险 | 说明 | 影响 |
|------|------|------|
| **代码全在 ui.js** | 1220 行单文件，维护成本上升 | 后续功能建议拆分 |
| **无 HTTPS** | HTTP 环境，`crypto.subtle` 不可用 | 已通过服务端登录绕过 |
| **API 无认证中间件** | 无 JWT/session，API 可被直接调用 | 数据安全性一般 |
| **无数据库备份** | 无自动备份脚本 | 需手动 pg_dump |
| **版本号手动管理** | `index.html` 中 `?v=N` 需手动递增 | 容易遗漏 |




### 4.2 提交流程

```bash
# 1. 查看当前改动
git status
git diff

# 2. 暂存指定文件（不要用 git add -A，避免混入无关文件）
git add "00_工作台/运营数据管理/xxx.js"

# 3. 提交（第一行简短标题，空行后详细说明）
git commit -m "$(cat <<'EOF'
fix/feat/refactor/chore: 标题（不超过 70 字符）

详细的说明内容，解释为什么改、改了什么、注意事项
EOF
)"

# 4. 推送到远程
git push origin main
```

### 4.3 提交前缀规范

| 前缀 | 适用场景 |
|------|---------|
| `feat:` | 新功能 |
| `fix:` | Bug 修复 |
| `refactor:` | 重构 |
| `chore:` | 配置/部署/工具 |
| `test:` | 测试 |
