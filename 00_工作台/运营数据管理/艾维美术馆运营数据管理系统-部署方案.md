## 一、当前架构（实际运行）

```
用户浏览器 → http://122.51.56.50
                  │
            Nginx（:80）[Docker 容器]
```

## 三、Docker 服务管理

### 3.1 服务状态

```bash
# 查看所有运行中容器
cd /opt/aiwei && docker compose ps

# 输出示例：
# NAME            STATUS                    PORTS
# aiwei-api-1     Up 1 hour                 3000/tcp
# aiwei-db-1      Up 1 hour (healthy)       5432/tcp
# aiwei-nginx-1   Up 1 hour                 0.0.0.0:80->80/tcp
```

### 3.2 日志

```bash
docker compose logs -f          # 全部容器日志
docker compose logs -f api      # 仅 API 日志
docker compose logs -f nginx    # 仅 Nginx 日志
```

### 3.3 启停

```bash
docker compose restart          # 重启全部
docker compose restart nginx    # 重启单个
docker compose down             # 停止全部（数据不丢失，卷保留）
docker compose up -d            # 启动全部
```

### 3.4 更新 API（server.js 修改后）

```bash
# 第 1 步：上传新文件
scp server.js root@122.51.56.50:/opt/aiwei/

# 第 2 步：远程重建并重启
ssh root@122.51.56.50 "cd /opt/aiwei && docker compose build api && docker compose up -d api"
```

### 3.5 更新前端（app/ 修改后）

```bash
# 前端文件直接挂载到 Nginx 容器，上传即生效
scp app/js/xxx.js root@122.51.56.50:/opt/aiwei/app/js/
# 重要：如果改 auth.js 等核心文件，记得递增 index.html 中 ?v=N 版本号
```

### 3.6 数据库操作

```bash
# 连接数据库
docker compose exec db psql -U postgres

# 查看所有表
\d

# 备份数据库
docker compose exec db pg_dump -U postgres > backup_$(date +%Y%m%d).sql

# 恢复数据库（先 docker compose down，删除 db-data 卷）
# 注意：这会清空数据，谨慎操作
```

## 四、API 文档

### 4.1 基础 URL

所有 API 通过 Nginx 反向代理访问：
```
http://122.51.56.50/rest/v1/{table}
```

### 4.2 端点和方法

| 方法 | 路径 | 说明 | 示例 |
|------|------|------|------|
| GET | `/rest/v1/{table}` | 列表查询 | `?order=date.desc&limit=10` |
| GET | `/rest/v1/{table}?id=eq.{id}` | 单条查询 | `GET /rest/v1/revenue?id=eq.xxx` |
| POST | `/rest/v1/{table}` | 新增 | body: `{"date":"2026-07-04","ticketQty":10}` |
| PATCH | `/rest/v1/{table}?id=eq.{id}` | 更新 | body: `{"ticketQty":12}` |
| DELETE | `/rest/v1/{table}?id=eq.{id}` | 删除 | — |
| POST | `/rest/v1/login` | 登录 | body: `{"username":"admin","password":"admin888"}` |
| POST | `/rest/v1/change-password` | 改密 | body: `{"userId":"xxx","newPassword":"xxx"}` |

### 4.3 过滤器

| 操作符 | 说明 | 示例 |
|--------|------|------|
| `eq.` | 等于 | `?project_name=eq.某某展` |
| `neq.` | 不等于 | `?id=neq.xxx` |
| `gte.` | 大于等于 | `?date=gte.2026-01-01` |
| `lte.` | 小于等于 | `?date=lte.2026-06-30` |
| `ilike.` | 模糊匹配 | `?project_name=ilike.%展览%` |

### 4.4 支持的表

`revenue`, `expense`, `space_usage`, `gallery_sales`, `app_config`, `users`, `operation_logs`, `project_registry`, `inventory`, `artworks`, `partners`, `content_posts`

### 4.5 字段命名

- 前端发送 JSON 使用 **camelCase**（如 `projectName`）
- 后端自动转换为 **snake_case**（如 `project_name`）存入 PostgreSQL
- 查询返回时自动转回 camelCase

## 六、首次部署流程（deploy.sh）

1. 打包 `Dockerfile` / `docker-compose.yml` / `nginx.conf` / `server.js` / `app/` 等文件
2. 上传 tar 包到服务器 `/tmp/`
3. 远程执行 Docker 安装（如未安装）、构建镜像、启动容器
4. 健康检查
5. 配置防火墙

**注意：** deploy.sh 中的 `DB_PASSWORD` 硬编码为 `Aiwei2024Gallery!`，如需修改请同时更改 docker-compose.yml 和 server.js 中的默认密码。

## 七、故障排查

### 7.1 API 返回 502

检查 API 容器是否在运行、日志是否有报错：

```bash
docker compose ps
docker compose logs api
```

常见原因：server.js 代码错误导致 Node.js 进程退出。

### 7.2 Nginx 端口冲突

```bash
# 检查端口占用
ss -tlnp | grep :80
# 如果被系统 Nginx 占用：systemctl stop nginx && systemctl disable nginx
```

### 7.3 数据库连不上

```bash
docker compose exec db pg_isready -U postgres
# 查看数据库日志：docker compose logs db
```

### 7.4 前端页面不更新

浏览器缓存问题。强制刷新（Ctrl+F5）或递增 `index.html` 中 JS 文件的版本号。

### 7.5 登录失败

- 确认 API 运行正常：`curl -s -X POST http://localhost/rest/v1/login -H 'Content-Type: application/json' -d '{"username":"admin","password":"admin888"}'`
- 数据库中的 SHA-256 哈希必须和密码匹配（用 Node.js `crypto.createHash('sha256').update(password).digest('hex')` 验证）

## 八、安全

| 风险 | 说明 | 缓解措施 |
|------|------|---------|
| HTTP（无 HTTPS） | 密码明文传输 | 当前内部使用，后续考虑 Nginx HTTPS |
| API 无认证 | 任何人可调用 API | 当前通过不公开 API 地址缓解 |
| 密码硬编码 | DB_PASSWORD 在 deploy.sh 中明文 | 生产环境应改为环境变量 |
| 无备份 | 数据库无自动备份 | 手动 pg_dump 备份 |
