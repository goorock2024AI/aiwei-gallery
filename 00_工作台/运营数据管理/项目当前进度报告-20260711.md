## 三、第十二期「空间使用重构」内容明细

### 核心改动

- **DB**：`space_usage` 加 `expected_payment_date` 字段；新建 `space_payments` 子表（外键 ON DELETE CASCADE）；新建 `space_usage_with_payments` 视图（`json_agg` 拼好子表数组 + `received_total` 聚合）
- **API**：`server.js` 加空间使用白名单 + `check-conflict` 端点（硬性阻止）+ `tableMap` 视图只读守卫 + 修复 5 处 `req.on('data')` 中文乱码
- **前端**：4 张财务卡（应收/已收/未收/逾期）+ 甘特图自绘（7 空间 × 31 天）+ 子表 payment 录入 UI + 冲突硬性阻止 + 8 处 `receivedAmount` → `receivedTotal`

### 关键技术坑

- **Buffer.toString('utf8') 中文乱码**：5 处 `body += chunk` 改为 `body += chunk.toString('utf8')`，因为 Node 默认 chunk 是 Buffer，字符串拼接时按 latin1 解码导致乱码
- **PG `json_agg` 视图**：用 `json_build_object` 拼好每个 payment 对象，前端 GET 一次拿全主表 + 子表 + 聚合

## 五、第十四期「画廊作品图片 + 导入」内容明细

### 核心改动

- **DB**：`artworks` 加 `image_url` 字段（TEXT，存 `/uploads/artworks/xxx.png` 相对路径）
- **后端上传**：`POST /rest/v1/artworks/upload` 接收 multipart/form-data，写到 `/uploads/artworks/`（容器内），文件命名 `时间戳_2字节随机.扩展名`，5MB 上限
- **Nginx 静态托管**：`location /uploads/` 用 alias 指向 `/var/cache/nginx/uploads/`（writable），通过 docker volume `uploads-data` 共享
- **前端 modal 加图片**：缩略图预览（120×120）+ file input 上传 + URL 输入框（备选）+ 移除按钮，编辑时实时预览
- **列表缩略图列**：60×60 圆角，自动 resolve 完整 URL（`/uploads/...` → `http://...`）
- **导入**：`_importArtworks` 支持 CSV/XLSX，含「图片URL」列，9 字段映射
- **模板**：`_downloadArtworkTemplate` 下载 CSV，列头含图片URL

### 关键技术坑（本次踩了 2 个新坑 + 1 个已知坑）

1. **nginx `/uploads/` 不能挂在只读 `/usr/share/nginx/html/`**：容器 rootfs 是 overlayfs readonly → 改用 `/var/cache/nginx/uploads/` + alias 解决
2. **nginx `proxy_pass http://api:3000/` 末尾的 `/` 会剥离 location 前缀**：`/rest/v1/artworks` → `/v1/artworks` 传给后端 → server.js 路径错位 → 404（这个 bug 之前一直存在但未触发）
3. **TABLE_COLS 白名单遗漏**：POST artworks 返回 404 + `{"error":"Not found"}` 而非 500，看不到错误。修复：新增表必须三件事（建表 + tableMap + 白名单）齐全
4. **Windows bash + curl 默认 GBK 编码**（第二次踩）：详见 memory `debug_curl_windows_gbk_encoding`
