## 一、项目背景与目标

### 1.1 背景

艾维美术馆日常经营涉及门票销售、咖啡套票、工坊体验、文创零售、场地租赁、画廊销售等多条收入线，以及相应的备用金支出管理。此前主要依赖 Excel 手工记账 + localStorage 本地存储的方式管理，存在以下问题：

- **多人无法同时使用**：Excel 文件只能单人操作，数据不能实时共享
- **数据孤岛**：收入、支出、空间使用数据分散在不同文件，难以关联分析
- **录入效率低**：表单式录入在前台收银场景中操作繁琐
- **管理层无法实时掌握**：经营数据汇总依赖人工整理，不能随时查看

### 1.2 核心目标

> 替代 Excel 手工记账，实现多人同时在线使用，快速完成运营数据的录入、分析、整理和分享，让管理层实时掌握经营数据。

### 1.3 系统定位

- **对内管理系统**（初期），不对外开放注册
- **未来可能**扩展为对外开放售票/预约的线上渠道

## 二、目标用户

| 用户角色 | 核心诉求 | 使用场景 |
|----------|----------|----------|
| **美术馆运营人员**（前台/运营） | 快速录入收入、记录支出、登记空间使用 | 日常前台收银、运营记账 |
| **美术馆管理层** | 实时查看经营概况、收入趋势、对比分析 | 经营决策、业绩回顾 |
| **财务** | 核对收款、导出数据、审核支出 | 月度对账、财务报表 |

### 权限策略（MVP）

- **已实现「记录登录账号」**（账号登录系统，2026-06-30 完成）
- 所有用户可查看、录入、编辑、删除所有数据
- 通过「记录登录账号 + 操作日志」保留操作追溯能力（操作日志 P1 未实现）

## 三、功能模块

### 3.1 首页概览（Dashboard）

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 月度核心指标 | 当月总收入、总支出、备用金余额、空间使用数量 | MVP |
| 近7日收入趋势图 | 折线图展示近7天每日总收入 | MVP |
| 近7日收入明细 | 最近10条收入流水列表 | MVP |

### 3.2 收入录入（POS 收银模式）⭐ 核心模块

**当前 MVP 已有功能：**

| 功能 | 说明 |
|------|------|
| 票务销售 | 动态票种按钮（加减数量），支持多种票种（普通票/套票等） |
| 咖啡销售 | 动态咖啡按钮（加减数量） |
| 工坊项目 | 下拉选择项目 → 填写数量/折扣 → 添加至清单 |
| 文创零售 | 手动输入单价/数量/产品名 → 添加至清单 |
| 其他收入 | 自定义金额 + 说明 |
| 收款方式选择 | 现金 / 扫码支付 / 对公转账 三选一 |
| 实时汇总 | 底部固定结算栏实时显示各分类小计和合计 |
| 编辑/删除 | 历史记录支持编辑和删除 |
| 关联项目/活动 | 可关联散客参观、展览名称、团建活动等 |

**未来规划：**

| 功能 | 优先级 | 说明 |
|------|--------|------|
| **画廊销售独立录入** | **P0** | 低频但有独立业务逻辑，需独立标签页 |
| **场地租金独立录入** | **P0** | 低频但有独立业务逻辑，需独立标签页 |
| 退款/作废操作 | P0 | 实际运营中退款场景频繁，支持全额/部分退款并记录原因 |
| 扫码枪支持 | P1 | 扫描商品条码快速录入 |
| 散客/团队票区分 | P1 | 不同价格策略 |

## 四、技术架构

### 4.1 总体架构

```
┌─────────────────────────────────┐
│  前端：纯 HTML/CSS/JS (无框架)   │
│  Nginx/IIS 部署 → 浏览器访问    │
├─────────────────────────────────┤
│  数据层：Supabase (PostgreSQL)  │
│  - revenue 表                   │
│  - expense 表                   │
│  - space_usage 表               │
│  - app_config 表                │
│  - users 表 (P1)                │
│  - operation_logs 表 (P1)       │
└─────────────────────────────────┘
```

### 4.2 技术选型

| 层级 | 选型 | 说明 |
|------|------|------|
| 前端框架 | 无框架（原生 HTML/CSS/JS） | 轻量、无需构建、方便维护 |
| 图表库 | Chart.js | 当前用 CDN 引入 |
| 数据库 | PostgreSQL（Supabase 托管） | 云端托管，免运维 |
| 部署 | Nginx / IIS | 前端部署到正式 Web 服务器，方便内部多人访问 |

### 4.3 数据模型

**revenue 表（收入流水）：**
- id, date, ticket_items(JSONB), ticket_amount, coffee_items(JSONB), coffee_amount, combo_qty, combo_amount, workshop_items(JSONB), workshop_amount, retail_items(JSONB), retail_amount, creative_amount, venue_amount, other_amount, other_desc, payment_method, cash_amount, account_amount, project_name, handler, notes, created_at

**expense 表（支出流水）：**
- id, date, type, project, category, amount, description, handler, invoice_status, receipt_status, related_activity, created_at

**space_usage 表（空间使用）：**
- id, date, space, project_name, type, client, status, receivable_amount, received_amount, notes, created_at

**app_config 表（动态配置）：**
- key (PK), value (JSONB), updated_at

**operation_logs 表（P1 — 操作日志）：**
- id, user_id, user_name, action_type (create/update/delete), table_name, record_id, old_values (JSONB), new_values (JSONB), ip_address, created_at

**users 表（已实现）：**
- id, username, password_hash (SHA-256), display_name, role (admin/editor/viewer), is_active, created_at, last_login_at
- RLS：允许 anon SELECT/INSERT/UPDATE
